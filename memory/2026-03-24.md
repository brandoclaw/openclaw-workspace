## Lessons Learned

- `channels.discord.accounts.*` in `openclaw.json` does **not** support an `allowlist` key — attempting to add it crashes the gateway. There is no per-account allowlist mechanism via config. Cross-bot delivery routing is the only workaround.
- When a channel is `groupPolicy: "allowlist"`, it can only send to explicitly permitted targets. DMs are not auto-permitted — the bot must be configured with proper channel-level access, not just an allowlist entry.
